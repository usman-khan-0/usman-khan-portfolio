│ 1. PROJECT OVERVIEW                                                          │
└────────────────────────────────────────────────────────────────────────────┘

This is a modern portfolio website for Muhammad Usman Khan that combines:
- Static frontend (HTML/CSS/JavaScript)
- AI-powered chatbot backend (Python + Vercel Serverless)
- LLM integration using Groq API (free)


┌────────────────────────────────────────────────────────────────────────────┐
│ 2. FRONTEND ARCHITECTURE                                                     │
└────────────────────────────────────────────────────────────────────────────┘

TECHNOLOGY STACK:
  ├─ HTML5 (Single Page Application)
  ├─ CSS3 (Custom styling with animations)
  ├─ JavaScript (Vanilla ES6+)
  └─ No frontend framework (React, Vue, etc.)

KEY FEATURES:
  ├─ Responsive design (mobile-first with vertical navigation)
  ├─ Fixed sidebar navigation
  ├─ Hash-based routing (#home, #about, #skills, etc.)
  ├─ Animated progress bars and circular indicators
  ├─ Modal dialogs (skill details, certificates)
  ├─ Contact form with Google Sheets integration
  ├─ Floating chatbot widget

MAIN FILES:
  ├─ index.html (4,413 lines - complete frontend)
  │   ├─ Embedded CSS (<style> tags)
  │   ├─ Embedded JavaScript (<script> tags)
  │   └─ All sections in one file
  └─ Images and assets in "images folder" directory

SECTIONS:
  ├─ #home      - Hero section with introduction
  ├─ #about     - About and background
  ├─ #skills    - Technical skills (5 tabs)
  ├─ #hobbies   - Interests and hobbies
  ├─ #stats     - Achievements and statistics
  ├─ #projects  - Project portfolio showcase
  ├─ #cv        - CV/Resume download section
  ├─ #certificates - Certifications gallery
  ├─ #learning-journey - Learning milestones
  └─ #contact   - Contact form

EXTERNAL LIBRARIES:
  ├─ Google Fonts (Poppins font family)
  └─ Font Awesome 6.0.0 (icons)

COLOR SCHEME:
  ├─ Primary:    #006400 (Green)
  ├─ Secondary:  #008000 (Green)
  ├─ Accent:     #ff0000 (Red)
  └─ Dark theme with light text


┌────────────────────────────────────────────────────────────────────────────┐
│ 3. BACKEND ARCHITECTURE                                                      │
└────────────────────────────────────────────────────────────────────────────┘

TECHNOLOGY STACK:
  ├─ Python 3.13
  ├─ Flask 3.0.0 (HTTP server framework)
  ├─ OpenAI SDK 1.6.0
  └─ Vercel Serverless Functions

BACKEND COMPONENTS:

  1. API Handler (api/index.py)
     ├─ Vercel serverless function
     ├─ Implements BaseHTTPRequestHandler
     ├─ Endpoints:
     │   ├─ POST /api/chat - Chatbot message endpoint
     │   ├─ GET /api/chat  - Health check
     │   └─ OPTIONS /api/chat - CORS preflight
     ├─ LLM Provider: Groq API (free)
     ├─ Model: Llama 3.1 8B Instant
     └─ Max tokens: 500 per response

  2. Local Development Server (server.py)
     ├─ Port: 8000
     ├─ Serves static HTML files
     ├─ Chatbot endpoint with CORS support
     └─ For local testing only

  3. Chatbot Implementation (chatbot.py, main.py)
     ├─ Uses OpenAI Agents framework
     ├─ Portfolio knowledge base in system prompt
     ├─ Guardrails to only answer about Usman's portfolio
     ├─ Conversation history support
     └─ CLI interface for testing

EXTERNAL API INTEGRATION:
  ├─ Groq API (https://api.groq.com/openai/v1)
  ├─ Free LLM inference
  ├─ Requires GROQ_API_KEY environment variable
  └─ Model: llama-3.1-8b-instant

DATA FLOW:
  1. User sends message via chatbot widget
  2. JavaScript POSTs to /api/chat
  3. Backend processes with Groq LLM
  4. Response sent back to frontend
  5. JavaScript displays message

│ 6. CHANGES NEEDED FOR VERCEL DEPLOYMENT                                     │
└────────────────────────────────────────────────────────────────────────────┘

CRITICAL ISSUES TO FIX:

  1. FOLDER NAMES WITH SPACES (HIGH PRIORITY)
     CURRENT PROBLEM:
       - "images folder" has a space in the name
       - "Projects Details" has a space in the name
       - Spaces in URLs are encoded as %20 which causes routing issues

     SOLUTION:
       - Rename "images folder" to "images" or "images_folder"
       - Rename "Projects Details" to "projects_details" or "project-details"
       - Update all references in index.html to new folder names

  2. VERCEL.JSON ROUTING CONFIGURATION
     CURRENT ISSUE:
       - The fallback route at the end only catches "/"
       - It doesn't catch paths like "/#about" or "/skills"
       - This causes 404 errors when navigating directly to sections

     SOLUTION:
       - Change the fallback route pattern from "/" to "/(.*)"
       - This ensures all non-matching routes fall back to index.html
       - See updated configuration below

  3. ENVIRONMENT VARIABLES
     REQUIRED ACTION:
       - Add GROQ_API_KEY to Vercel Environment Variables
       - Get free API key from: https://console.groq.com/
       - Add in Vercel Dashboard: Settings → Environment Variables

  4. FILE STRUCTURE VERIFICATION
     REQUIRED FILES:
       - index.html must be in the ROOT directory
       - api/index.py must exist for chatbot functionality
       - api/requirements.txt must include openai==1.6.0

  5. BUILD OUTPUT DIRECTORY
     NOTE:
       - Since this is a static site, no build process is needed
       - Vercel automatically serves index.html from root
       - API functions are built automatically from /api directory


┌────────────────────────────────────────────────────────────────────────────┐
│ 7. 404 ERROR SOLUTION                                                         │
└────────────────────────────────────────────────────────────────────────────┘

ROOT CAUSE:
  The vercel.json routing configuration has an incomplete fallback route that only
  catches "/" path, but doesn't handle:
  - Hash routes like "/#about"
  - Direct navigation to sections
  - Client-side routing paths

SOLUTION STEPS:

  STEP 1: Update vercel.json
  Replace the entire vercel.json file with the corrected version below.
  (See section 8 for the complete updated file)

  STEP 2: Rename folders with spaces
  A. Rename "images folder" to "images" or "images_folder"
  B. Rename "Projects Details" to "projects" or "project-details"
  C. Update index.html to use new folder names:
     - Find: images folder/ → Replace with: images/
     - Find: Projects Details/ → Replace with: projects/

  STEP 3: Verify file locations
  - index.html must be in the project root (not in a subfolder)
  - api/index.py must exist in the api directory
  - All assets must be correctly referenced

  STEP 4: Redeploy to Vercel
  A. Commit changes to git
  B. Push to GitHub
  C. Vercel will auto-deploy
  D. Or manually deploy: vercel --prod

  STEP 5: Add Environment Variable
  - Go to Vercel Dashboard → Project → Settings → Environment Variables
  - Add: GROQ_API_KEY = [your_groq_api_key]
  - Select: Production, Preview, Development
  - Save and redeploy
